How HTML Accesses CSS

First of all, CSS stands for Cascading style sheets. It “is a style sheet language used for describing the presentation of a document written in a markup language.” - Wiki

CSS is used to style web pages and it is used along side with HTML. Every Web page uses CSS because of ease of use. It can be implemented in either of three easy methods:

  • External
  • Internal
  • Inline

    • Inline CSS

    An inline style can be used if a style is to be applied to one element/event. Inline styles are quick and easy to apply. It is written directly in the “h1” of the code, and it overwrites any internal and external techniques. This is what makes it so quick and easy, but it has to be re-applied to every single event you want to use it in. E.g h1 style="color:red;

    Advantages:

  • It is quick and easy to use if you are changing a single element.
  • Saves the effort of creating an external file.
  • It is also good for testing. This is because it is easier to correct lines of code from the source rather than changing the source
  • It also loads quicker over the internet since it is all coming from one file. (lower http requests).
  • It can be used to debug pages that have errored.
    • Disadvantages:
  • If Inline would be the only thing to be used, every single element would have to have its own code, and for big websites, that just is not efficient enough. For example, if you wanted all the text to be bold, you would have to add the command to each p/p tag on the document.
  • Moreover, it will overwrite any other implementations of CSS since it is the most direct form of css. This could act as an annoyance to the designer.

    • Read More

    Internal CSS

    Internal codes are embedded within the head.../head tag. Internal CSS dominates the rules of an external style sheet. So it can be used for a document with one single style. A style tag is used within the head tag.

    Advantages:

  • Internal CSS can be used to implement a specific style to the whole document with just one entry. E.g. If the designer wants generally wants all text to be one colour, then he can add a tag in the internal section of the document and that would be all he would need to do.
    • Disadvantages:
  • Internal CSS loads very slowly due to low http requests. This can detract users from any website, since the average wait time is as low as 2 seconds.

    • Read More

    External CSS

    External css is a separate document of css that is applied to multiple pages of HTML. It is saved with a .css file extension. It is then referenced using a link within the head, instead of using style. This means an entire website can be changed with the modification of one small file.

    Advantages:

  • The file size of the HTML document is smaller since all the coding to do with styling is on another document.
  • It is easier for developers to read and work with the coding because the HTML sheet is separated from the styling sheet.
  • It also makes it so much easier to reuse your styling code, since it is stored on a separate sheet and can be copied onto other websites.
  • It can also allow for a higher number of visitors, since it is more optimised for search engines.
    • Disadvantages:
  • It is not as useful for small websites, since they can get overloaded with information when a page gets deleted. This is because the code stays there when a page is deleted.

    The Box Model

    In css, the box model is used to determine the design and layout of a HTML page. It usually consists of four elements:

  • Margin
  • Border
  • Padding
  • Content

    • The box model is a set of standards for how web browsers will display any content on a webpage.

    Margin

    The margin is a transparent section of the box model and it clears the area around the border. Margins are able to collapse into one another so that each page blends nicely.

    Border

    The border goes around the padding and content. They can be either coloured or transparent. Borders are usually used for decorative purposes.

    Padding

    The padding is also transparent and it clears an area around the content. It is the same colour as the background colour that is being used for the box.

    Content

    The content is where the images and text appear on the webpage. It is the main element of the box. 
    Here is an example of a box model:
div {
	width: 300px;
	border: 25px solid green;
	padding: 25px;
	margin: 25px;
}

However, if just HTML was used, web pages would look very boring and lifeless even though it would be displaying the important information. HTML combined with the use of CSS makes web pages look a lot more aesthetically nice and is what we usually see on web pages on the internet. CSS need scripting language.

  • JavaScript
  • Validation
  • Optimisation
  • Animation
  • Security

    • Javascript

    Javascript is the programming language of HTML. It controls the behaviour of the web pages. So along with HTML and CSS, Javascript is definitely a fundamental language to learn if you want to create and modify web pages. It is quite an easy language to learn and helps users to branch to harder and more specific languages too. So there are three languages that need to be learned by a any web developer:
  • HTML
  • CSS

    • This is because HTML is used to define what content is on a web page, CSS is used to determine the style, look and layout of a web page and JavaScript is used to program how the website works.

    How Scripting Languages Improve Functionality

    A Scripting Language is a tool used to fulfill relatively simple tasks such as creating interactive functions for the user. It is not as good as full programming languages, but it doesn’t always need to be for what it is used for. As i’ve mentioned, HTML is used to display content and define it on a webpage, but Javascript or php may be used to make it more of a dynamic webpage. JavaScript is what makes the website functional and interactive for users. These scripting languages can be implemented into a html file so as to create interactive menus for users and so on.


    The use of a scripting language is that it makes websites more interactive because the page will respond to the users actions instantly. JavaScript is the main client side scripting language of the web, and if a website uses JavaScript effectively, then it can add user prompts, navigation assistance and other kinds of client side calculations. It can make web pages more user friendly once the page has been delivered to a user. It is important to have recent browsers though, such as Google Chrome and FireFox, since scripts may not work properly on outdated browsers.


    Front-end Context

    Front-end context uses scripting through the user interface. For example, this is how a web page may resize information and boxes to fit the interface of the user, whether it is mobile or desktop.

    Client-side Calculations

    Client-side calculations are where any interactive scripting language commands are executed on the user's computer, since the calculations are done from the user’s side instead of the servers side.

    How Web Pages using Scripts are Implemented into Different Browsers


    JavaScript will display differently on each browser, since each browser is usually programmed to deal with the javascript in a different manor. With there being many different kinds of browsers on the market, it is wise to test and make sure each browser deals with the scripting how you would want it to.

    To test this efficiently, it would be wise to check what the most used browsers currently are and how they would deal with code. It would also be a smart idea to make sure the code works best for the most used browser. Here is a chart below to show this information:

    Browser Detection

    As you can see on the table above, Chrome is the most used browser by far, and its popularity is rising every month. So it would be smart to base your scripting to work off of chrome, but still so that it would work on these other browsers. To do this we need to make all scripting display the same; or as close to being the same as possible. This is called Browser Detection.

    Browser Detection is very important for developers, since it allows the developer to see how optimal the scripting is for all browsers, and not just one of them. The detection process is to make sure that the right code is used for the browser in use to allow the website to work at its best. All browsers are programmed differently which means that client side scripts would be shown differently on a client’s browser. CSS code should all look the same on all browsers but with JavaScript it can be wrong. This is why it has to be tested and told how to look.

    Object Detection

    Object Detection works in a similar way, but instead of just showing what it would look like on each browser, it reads which browser and version is being used and it adds code to make the script compatible with the browser. It may check if the script is compatible or not, and if it is not, it simply will not go through with the process.
